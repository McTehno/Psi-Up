# Besednjak pojmov za projekt priporočilnega sistema učnih poti

### 1. Osnovni pojmi projekta

| Pojem | Razlaga |
| :--- | :--- |
| **Priporočilni sistem** | Sistem, ki uporabniku na podlagi njegovih odgovorov, predznanja, želja in ciljev predlaga ustrezno učno pot. V našem projektu priporočilni sistem ne predlaga posameznih učnih enot neposredno, ampak predvsem primerne module oziroma curriculum. |
| **Učna pot** | Zaporedje modulov in učnih enot, ki uporabnika vodijo od trenutnega znanja do želenega cilja. Učna pot je lahko prilagojena posamezniku ali podjetju. |
| **Curriculum** | Celoten izobraževalni načrt, sestavljen iz več modulov. Predstavlja širšo pot do določenega končnega cilja, na primer “osnovna digitalna pismenost”, “podatkovni analitik” ali “uporabnik naprednih digitalnih orodij”. |
| **Modul** | Vsebinsko zaokrožena enota znotraj učne poti. Modul je sestavljen iz več učnih enot in je povezan z eno ali več digitalnimi kompetencami. Primer: “Osnovna uporaba Excela”, “Napredna podatkovna analitika”, “Varnost pri uporabi interneta”. |
| **Učna enota** | Manjši del modula, ki pokriva konkretno vsebino ali temo. Lahko je lekcija, predmet, vaja, delavnica ali krajši sklop znanja. Učne enote bodo podane s strani stranke oziroma naročnika. |
| **Obvezna učna enota** | Učna enota, ki jo mora uporabnik opraviti, da uspešno zaključi modul. |
| **Izbirna učna enota** | Učna enota, ki ni nujna, ampak uporabniku omogoča poglobitev, specializacijo ali dodatno znanje. |
| **Predpogoj** | Znanje, kompetenca, modul ali učna enota, ki jo mora uporabnik poznati oziroma opraviti, preden lahko nadaljuje z zahtevnejšo vsebino. Primer: za “napredno analitiko v Excelu” je predpogoj “osnovna uporaba Excela”. |
| **Končni cilj** | Cilj, ki ga želi uporabnik doseči z učno potjo. Lahko je vezan na kompetenco, raven znanja ali vlogo, ki jo bo uporabnik po zaključku sposoben opravljati. |
| **Vloga** | Praktični profil ali namen, za katerega se uporabnik usposablja. Primer: “digitalno pismen uporabnik”, “podatkovni analitik”, “uporabnik AI orodij”, “oseba, ki zna varno uporabljati spletne storitve”. |

## 2. Pojmi po DigComp standardu

| Pojem | Razlaga |
| :--- | :--- |
| **DigComp** | Evropski okvir digitalnih kompetenc za državljane. Uporablja se za opis, ocenjevanje in razvoj digitalnih kompetenc posameznikov. |
| **Digitalna kompetenca** | Kombinacija znanja, spretnosti in stališč, ki posamezniku omogočajo varno, kritično, ustvarjalno in učinkovito uporabo digitalnih tehnologij. |
| **Kompetenčno področje** | Večja skupina sorodnih digitalnih kompetenc. DigComp tradicionalno zajema pet glavnih področij digitalnih kompetenc. |
| **Kompetenca** | Konkretna sposobnost znotraj kompetenčnega področja. Primer: iskanje informacij, varovanje naprav, ustvarjanje digitalnih vsebin, reševanje tehničnih težav. |
| **Raven kompetence** | Stopnja, na kateri uporabnik obvlada določeno kompetenco. Lahko je osnovna, srednja, napredna ali zelo napredna. V DigComp se ravni uporabljajo za opis napredovanja od začetnika do visoko specializiranega uporabnika. |
| **Znanje** | Teoretično razumevanje neke teme. Primer: uporabnik razume, kaj je močno geslo ali kaj pomeni varovanje osebnih podatkov. |
| **Spretnost** | Praktična sposobnost uporabe znanja. Primer: uporabnik zna ustvariti varno geslo, poslati e-pošto, uporabiti Excel ali poiskati zanesljive informacije na spletu. |
| **Stališče** | Način razmišljanja, odnos ali odgovorno vedenje pri uporabi digitalnih tehnologij. Primer: uporabnik je pozoren na varnost, zasebnost, zanesljivost virov in etično uporabo AI orodij. |

## 3. Glavna kompetenčna področja DigComp

*DigComp ima tri osnovne gradnike: področja, kompetence in ravni usposobljenosti; spodaj je pet ključnih področij:*

| Področje | Razlaga v kontekstu projekta |
| :--- | :--- |
| **Informacijska in podatkovna pismenost** | Sposobnost iskanja, razumevanja, vrednotenja, organiziranja in uporabe informacij ter podatkov. Primeri: iskanje informacij na spletu, ocenjevanje zanesljivosti virov, osnovna uporaba podatkov v Excelu. |
| **Komuniciranje in sodelovanje** | Sposobnost uporabe digitalnih orodij za komunikacijo, sodelovanje in deljenje informacij. Primeri: e-pošta, Teams, Zoom, spletni bonton, sodelovanje v skupnih dokumentih. |
| **Ustvarjanje digitalnih vsebin** | Sposobnost ustvarjanja, urejanja in prilagajanja digitalnih vsebin. Primeri: dokumenti, predstavitve, spletne strani, slike, uporaba AI orodij za pomoč pri ustvarjanju vsebin. |
| **Varnost** | Sposobnost varne uporabe digitalnih naprav, podatkov, identitete in spletnih storitev. Primeri: zaščita računalnika, varna gesla, prepoznavanje spletnih prevar, varovanje osebnih podatkov. |
| **Reševanje problemov** | Sposobnost prepoznavanja in reševanja tehničnih ali digitalnih težav ter izbire ustreznih digitalnih orodij. To vključuje razumevanje problema, iskanje rešitve in izbiro primernega orodja. |

## 4. Pojmi za ocenjevanje uporabnika

| Pojem | Razlaga |
| :--- | :--- |
| **Vprašalnik** | Nabor vprašanj, s katerimi sistem ugotovi uporabnikovo trenutno znanje, cilje, interese in potrebe. |
| **Samoocena** | Način ocenjevanja, pri katerem uporabnik sam pove, koliko zna ali kako samozavestno uporablja določeno orodje. |
| **Ocenjevanje digitalnih kompetenc** | Postopek, s katerim sistem ugotovi, katere digitalne kompetence uporabnik že ima in katere želi izboljšati. |
| **Trenutno znanje** | Znanje in spretnosti, ki jih uporabnik že ima pred začetkom učne poti. |
| **Želene kompetence** | Kompetence, ki jih uporabnik želi pridobiti (npr. "želim se naučiti uporabljati Excel"). |
| **Vrzel v znanju** | Razlika med trenutnim znanjem uporabnika in znanjem, ki ga potrebuje za dosego cilja. |
| **Začetna točka** | Mesto v učni poti, kjer uporabnik začne glede na svoje predznanje. |
| **Napredovanje** | Prikaz, kako uporabnik postopoma prehaja skozi module proti končnemu cilju. |

## 5. Pojmi za priporočanje učnih poti

| Pojem | Razlaga |
| :--- | :--- |
| **Personalizacija** | Prilagoditev učne poti posamezniku glede na njegovo znanje, cilje in želeno raven kompetenc. |
| **Prilagodljiva učna pot** | Učna pot, ki se lahko spremeni glede na uporabnikove odgovore ali napredek. |
| **Predpripravljena učna pot** | Vnaprej sestavljena učna pot za določen cilj (npr. "Osnovna digitalna pismenost"). |
| **Priporočilo** | Rezultat sistema, ki predlaga najbolj primerne module za uporabnika. |
| **Razlaga priporočila** | Kratka obrazložitev, zakaj je sistem predlagal določeno pot (npr. "Ker želite izboljšati Excel, vaše znanje pa je osnovno"). |
| **Ena priporočena pot** | Sistem prikaže samo eno, najbolj optimalno pot. |
| **Več možnih poti** | Sistem ponudi več alternativnih poti, med katerimi uporabnik izbira. |
| **Dummy curriculum** | Začasni ali testni curriculum, ki se uporablja za razvoj in prikaz prototipa. |

## 6. Pojmi za strukturo modulov

| Pojem | Razlaga |
| :--- | :--- |
| **Osnovni modul** | Namenjen uporabnikom z malo ali nič predznanja (npr. "Osnovna uporaba e-pošte"). |
| **Napredni modul** | Za uporabnike, ki želijo razviti zahtevnejše kompetence (npr. "Napredna analitika v Excelu"). |
| **Modul z več kompetencami** | Modul, ki pokriva več kompetenc hkrati (npr. "Uporaba GPT pri delu"). |
| **Povezava modul–kompetenca** | Relacija, ki določa, katere kompetence razvija določen modul. |
| **Povezava učna enota–kompetenca** | Relacija, ki določa, katere konkretne kompetence razvija posamezna učna enota. |
| **Težavnost** | Oznaka zahtevnosti (osnovna, srednja, napredna). |
| **Trajanje** | Predviden čas, potreben za zaključek enote ali modula. |
